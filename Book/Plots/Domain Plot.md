#GraphTheory #Definition 
# **Definition**

---

> [!Definition]
> A transfer plot, also called a domain or mapping plot, is a graph that represents a known functional relationship between an input domain and an output domain of a system. Each value of the independent variable maps to a specific, deterministic value of the dependent variable according to a defined model or equation. Transfer plots are visual representations of a system’s transfer function.

  ---
  
# **Key Ideas**

- Both axes represent physical [[domains |domain]]
    <br>

- The relationship is defined by theory, design, or validated modeling
    <br>
- Adjacent points are causally and mathematically related
    <br>
- Lines represent the system model, not just visual guidance
    <br>
- Commonly found on datasheets and system specifications
    

# **Examples:**

---

> [!example]
> A load cell datasheet shows output voltage versus applied force. The plotted line represents the transfer function of the sensor, allowing engineers to predict output voltage for any force within the rated range.


- Used for prediction and system design
    <br>
- Assumes linearity or explicitly defined nonlinearity
    

  ### Domain plots are used to:
+ Predict what output is generated from an input
+ Visualize design calculations
+ Control system development
+ Interoperate datasheet information
